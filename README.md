# Конфигурационное управление. Домашнее задание. Вариант 2

## 1.Общее описание

Разработан инструмент командной строки для визуализации графа зависимостей пакетов Java в формате Maven. Инструмент анализирует зависимости заданного пакета, строит полный граф зависимостей (с учетом транзитивности) и предоставляет различные способы его визуализации и анализа.

## Архитектура и модули

### Основные модули

#### 1. Модуль конфигурации (Configuration Module)
**Назначение**: Обработка параметров командной строки и валидация входных данных.

**Реализованные функции**:
- Парсинг аргументов командной строки
- Валидация формата имени пакета (groupId:artifactId)
- Проверка корректности версий
- Валидация URL репозиториев
- Проверка существования тестовых файлов
- Валидация форматов выходных файлов

**Ключевые классы и методы**:
- `DependencyVisualizer.parse_arguments()` - основной метод парсинга
- `DependencyVisualizer.validate_arguments()` - валидация всех параметров
- `argparse.ArgumentParser` - стандартный парсер аргументов Python

#### 2. Модуль сбора данных (Data Collection Module)
**Назначение**: Получение информации о зависимостях из различных источников.

**Реализованные функции**:
- Загрузка POM-файлов из Maven репозиториев
- Парсинг XML структуры POM-файлов
- Извлечение информации о зависимостях
- Работа с тестовыми файлами репозиториев
- Обработка сетевых ошибок и таймаутов

**Ключевые классы и методы**:
- `get_pom_url()` - формирование URL для POM-файла
- `download_pom()` - загрузка POM-файла из сети
- `parse_dependencies_from_pom()` - парсинг зависимостей из XML
- `get_dependencies_from_test_file()` - чтение тестовых данных

#### 3. Модуль анализа графов (Graph Analysis Module)
**Назначение**: Построение и анализ графов зависимостей.

**Планируемые функции**:
- Рекурсивный обход зависимостей (алгоритм DFS)
- Обнаружение циклических зависимостей
- Построение полного транзитивного замыкания
- Определение порядка топологической сортировки
- Расчет уровней зависимостей

#### 4. Модуль визуализации (Visualization Module)
**Назначение**: Преобразование графов зависимостей в графические форматы.

**Планируемые функции**:
- Генерация диаграмм на языке Mermaid
- Экспорт графов в форматы PNG/SVG
- Настройка стилей и форматов визуализации
- Поддержка различных схем окрашивания узлов

#### 5. Модуль тестирования (Testing Module)
**Назначение**: Обеспечение корректности работы инструмента.

**Реализованные функции**:
- Тестовый режим работы с файловыми репозиториями
- Формат тестовых файлов с простым синтаксисом
- Валидация тестовых данных
- Демонстрационные примеры работы

## Конфигурация и настройки

### Параметры командной строки

#### Обязательные параметры:
| Параметр | Короткая форма | Тип | Описание | Пример | Валидация |
|----------|----------------|-----|----------|--------|-----------|
| `--package` | `-p` | Строка | Имя анализируемого пакета в формате groupId:artifactId | `org.springframework:spring-core` | Должен содержать двоеточие, только строчные буквы, цифры, точки и дефисы |
| `--version` | `-v` | Строка | Версия анализируемого пакета | `5.3.0`, `1.0.0-SNAPSHOT` | Не пустая строка, соответствие семантическому версионированию |

#### Опциональные параметры:
| Параметр | Короткая форма | Тип | Описание | Значение по умолчанию | Валидация |
|----------|----------------|-----|----------|----------------------|-----------|
| `--url` | `-u` | Строка | Базовый URL Maven репозитория | `https://repo1.maven.org/maven2/` | Корректный URL, доступность не проверяется |
| `--test-mode` | `-t` | Флаг | Включение тестового режима работы | `False` | Только в сочетании с `--file` |
| `--file` | `-f` | Строка | Путь к файлу тестового репозитория | Нет | Должен существовать, читаемый формат |
| `--output` | `-o` | Строка | Имя выходного файла с изображением графа | `dependency_graph.png` | Корректное расширение (.png, .jpg, .svg) |
